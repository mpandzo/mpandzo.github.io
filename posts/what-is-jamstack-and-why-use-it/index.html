What is JAMStack and why use it?

I am a big proponent of JAMStack. With JAMStack, small teams can build interactive, performant, scalable and secure websites at low operating cost.

What is JAMStack?⌗

JAMstack stands for JavaScript, APIs, and Markup. The term was coined by Mathias Biilmann to describe “a modern web development architecture based on client-side JavaScript, reusable APIs, and prebuilt Markup”.

Where can static sites be hosted?⌗

The stack name itself was coined by the founders of netlify.com. Other static site hosts include surge.sh, vercel.com, render.com, pages.github.com to name a few.

Why use JAMStack?⌗

My proclivity to turn to this particular stack for many projects we undertake does us no harm in water cooler conversations with other developers that follow industry trends, however, our choice of JAMstack is not out of vanity. Most often it’s simply the optimal choice to make. With advanced frontend frameworks and a wide-array of available APIs, there is almost no limit to the type of website one can build with JAMStack.

Interaction and dynamic content⌗

Though markup is pre-build at deploy time into static files, a JAMStack website is far from interactively static. Frontend frameworks like React, Angular, Vue, et al. in conjunction with API communication can be used to build reactive and dynamic frontend user interfaces.

Performance⌗

Static assets and pre-built markup files are served over a CDN, greatly reducing time-to-first-byte and thus overall website loading time. Latency is further reduced due to files being served from servers geographically closer to clients’ geographical locations thanks to the distributed nature of CDNs.

Scalability⌗

Traffic increases or spikes are traditionally handled well by CDN infrastructure without the need of any further input from website administrators. It is also easy to distribute static files and replicate lambda functions across servers when needed.

Security⌗

JAMStack’s serverless nature implies there is less need to worry about conventional server and database security vulnerabilities. API calls are made to lambda functions which run in isolated environments with a greatly reduced attack surface area. CDNs also have inbuilt DDOS prevention and mitigation mechanisms.

Lower operating cost⌗

Hosting static files in the cloud is cheap. Lambda function costs are also based purely on consumed compute times. Compared to traditional web hosting solutions, JAMStack solutions on popular providers like Netlify, Surge, Vercel (Zeit) etc are comparably cheaper.
